Список продуктов хранится в БД (в моём случае это PostgreSQL).
По условию задачи (неавторизованный пользователь) хранить данные корзины постоянно нецелесообразно =>
хранение реализовано вс помощью модуля store, принцип действия которого аналогичен LocalStorage.
Корзина хранится всё время работы приложения, при остнове/рестарте данные удаляются.

Описание API

GET /api/products - получение списка продуктов
GET /api/cart - просмотр содержимого корзины
DELETE /api/cart/<id> - удаление из корзины продукта с заданным id
POST /api/cart - добавление товара в корзину

тело запроса:

{
    pool: {
        id: 3,                   // id товара
        quantity: 5              // количество единиц товара
    }
}
